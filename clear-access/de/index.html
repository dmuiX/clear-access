Was ich mache

Leistung

Ich richte einen klaren, zentralen Zugang ein. Eure Tools bleiben unverändert – geregelt wird nur der Zugang.

  • Ein zentraler Einstieg für self-hosted Tools
  • Ein Login für mehrere Tools
  • Klare Entscheidung, wer welches Tool erreichen darf
  • SaaS-Login über denselben Account, wenn unterstützt

Technischer Hintergrund

Tools
Technischen Hintergrund anzeigen

Ich setze eine zentrale Zugriffsschicht vor eure Tools. Sie entscheidet, wer welches Tool erreichen darf. Die Anwendungen bleiben unverändert — nur der Zugang wird konsistent.

  • Identity: Authelia mit OIDC/OAuth2 für SSO
  • Directory: Samba-DC / LDAP / AD, MS Entra ID, Google Workspace
  • Gateway: SWAG / Nginx Reverse Proxy + SSL
  • MFA: 2FA auf kritischen Tools
  • Access rules: klares Rollenmapping & Offboarding

Keine Migration von Nutzern oder Daten. Ziel ist ein klarer Zugangszustand — kein Umbau.
